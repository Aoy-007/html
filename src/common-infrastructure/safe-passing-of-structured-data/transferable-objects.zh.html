可传输对象

可传输对象 支持在 事件循环 之间传输。传输其实就是有效地重新创建对象并共享对底层数据的引用，然后将被传输的对象分离。 在转让昂贵资源的所有权时很有用。并非所有对象都是 可传输对象， 也并非 可传输对象 的所有方面都会在传输时被保留。

传输是不可逆转的、不幂等的操作。对象一旦被传输就不可再次传输或使用。

可传输的平台对象 有一个 [[Detached]] 内部槽，以及下列的内部方法：

[[Transfer]] ( targetRealm )

虽然所有 平台对象 都有 [[Clone]] 内部方法，但并非都有 [[Detached]] 内部槽以及 [[Transfer]] 内部方法。

可传输的平台对象 必须定义 [[Transfer]] 内部方法，它或者抛出一个异常，或者 在targetRealm创建一份this 的克隆并返回。 this与返回值共享底层数据。this的 [[Detached]] 内部槽的值设为 true。 具体如何传输取决于这些对象如何去定义它。

JavaScript 标准中定义的对象直接交由 StructuredCloneWithTransfer （技术上是 IsTransferable 和 Transfer）抽象操作。
