数字密集计算的后台 worker

workers 最简单的使用方式是在不打断 UI 的情况下执行计算密集型的任务。

本例子中，主文档启动（spawns ）了一个 worker 来（naïvely）计算质数， 然后逐渐地显示新发现的质数。
