没有 DOM 的内联内容的页面加载处理模型

当用户代理显示一个内联在 浏览上下文 中的用户代理页面时, 用户代理应该创建一个 Document 对象，把它标记为 HTML 文档，设置其 内容类型 为 "text/html"， 初始化 Document 对象， 然后或者把 Document 关联到一个自定义渲染过程（不适用正常的 Document 渲染规则） 或者把那个 Document 更改为用户代理希望渲染的内容。

一旦页面已经建立，用户代理必须表现地像已经 停止解析 一样。

在创建 Document 对象时，用户代理必须不参考清单，并传入新创建的 Document 来执行 应用缓存选择算法。

在创建 Document 对象后，页面结束完全建立之前， 用户代理必须 用新的页面更新会话历史。
